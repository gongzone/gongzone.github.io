## Recursion의 의미

한국말로는 재귀(再歸). 원래의 자리로 되돌아 온다는 뜻이다. ex) 재귀 본능. <br />
그럼 프로그래밍 관점에서는 무슨 의미일까?

**(1) Recursion은 자기 자신을 참조하여 자기 자신을 정의내리는 과정이다.**

말이 조금 어렵다. 조금 더 구체적으로 접근해보자.

**(2) Recursion은 자기 자신을 호출하는 함수가 행하는 일련의 과정을 말한다.**

자기 자신을 호출하는 함수를 우리는 **Recursive function** 이라고 한다. 이 함수가 수행하는 연속적인 과정이 바로 **Recursion**이다.

간단한 예를 들어보자.

<DecoDiv display="default">
  🤓 Sum of Natural Number
  <br />
  <br />✔ 1 + 2 + 3 + 4 ... + n = ?
</DecoDiv>

다음과 같은 문제가 있다고 한다. n 개의 연속적인 자연수가 주어진다고 할 때 그 합을 구하는 문제이다.

▶ Sum(n) = 1 + 2 + 3 + 4 ... + n 이 함수를 아래와 같이 표현 가능하다. <br />
▶ Sum(n) = 1 + 2 + 3 + 4 ... + (n - 1) + n

그렇다면 1 + 2 + 3 + 4 ... + (n - 1) 부분을 Sum(n - 1) 로도 표현 가능할 것이다.<br />
▶ Sum(n - 1) = 1 + 2 + 3 + 4 ... + (n - 1)

따라서 최종적으로 Sum(n)은 다음과 같이 생각할 수 있다. <br />
▶ Sum(n) = Sum(n - 1) + n

n 값에 5를 대입해보면 Sum(5)라는 함수는 1에서 4까지의 합(Sum(4))과 n 값인 5를 더한 결과를 도출해낸다. 위의 함수식에서 볼 수 있다 싶이

✅ **자기 자신(Sum(n - 1))**을 참조하여 **자기 자신(Sum(n))**을 정의내린다. -> (1)<br />

위의 함수를 코드로 작성해보자.

```cpp
int Sum(int n)
{
  if(n <= 1) //base condition
    return 1;
  else
    return Sum(n - 1) + n; //자기 자신을 호출
}
```

recursive function은 base condition이 필요하다. 그렇지 않으면 함수 안에서 자기 자신을 호출하는 행위가 무한정으로 발생하게 된다.
위의 base condition은 n의 값이(해당 매개변수) 1 일때 1을 리턴하는 조건이다.(Sum(1) 일 때 return 1) 즉 해당 매개변수인 n의 값이 1까지 도달하면
자기 자신을 호출하는 행위를 멈추고 특정 값을 리턴한다.

위 코드를 보면 알 수 있듯이

✅ **Recursive function은 자기 자신을 호출하는 함수**를 말하며 **Recursion은 그 함수가 행하는 일련의 과정**을 의미한다 -> (2)<br />

아직 우리에게는 물음이 존재한다. 그래서 저 일련의 과정이 무엇인가? 즉 저 코드가 어떻게 동작하는가?
recursive function의 동작을 이해하려면 **메인 메모리의 스택(stack)** 영역에 대한 이해가 필요하다.
그것을 기초로 우리는 저 코드를 분석할 수 있다.

그리고 가장 중요한 질문인 **'그래서 이걸 어따 쓰는데?'**가 남아있다.
